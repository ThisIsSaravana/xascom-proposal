# Comprehensive Project Plan: Insurance Broker CRM & Analytics Dashboard

## 1. System Architecture & Tech Stack
* **Frontend:** React or Next.js (Component-based, highly scalable).
* **UI/Charts:** Tailwind CSS combined with Tremor or Recharts for financial dashboards.
* **Backend/Database:** Supabase (PostgreSQL) - Excellent for relational data, handles authentication out-of-the-box, and supports real-time updates.
* **Data Parsing:** PapaParse (for CSV) and SheetJS (for Excel).
* **Security:** Strict Content Security Policy (CSP) to protect financial/commission data.

## 2. Database Schema (Relational Structure)
* **Users Table:** System users with roles (Admin/Management vs. Agent/Telecaller).
* **Providers Table:** List of insurance carriers they sell for (e.g., Star Health).
* **Customers Table:** Policyholder details (Name, Mobile, Email).
* **Policies Table:** Policy No, Provider ID, Product Type, Sum Insured, Gross Premium (Deal Size), Commission Amount, Expiry Date, Status (Pending, Renewed, Not Renewed).
* **Call Logs/Interactions Table:** Linked to the Policy/Customer. Stores Timestamp, Agent ID, Disposition (Answered, Call Back, Not Interested), Follow-up Date/Time, and strictly contextual text notes.

## 3. The "Hassle-Free" Data Ingestion Pipeline
Since the client receives data in various spreadsheet formats from different carriers, the ingestion process must be bulletproof.
* **Drag & Drop Interface:** Utilizing React Dropzone for file uploads.
* **The Staging Grid (Crucial Step):** Never insert directly into the DB. Parse the file on the frontend and display it in an editable grid.
* **Auto-Mapping:** Allow the admin to map spreadsheet columns (e.g., "GROSS") to database columns (e.g., "premium_amount").
* **Validation & Error Highlighting:** Flag missing data (e.g., missing policy numbers or phone numbers) in red. Let the admin fix it directly in the browser grid.
* **Bulk Assignment:** Allow the admin to assign batches of newly imported expiring policies to specific agents (e.g., assigning 50 rows to Priya) before hitting "Confirm Import."

## 4. Agent CRM Interface (The Daily Workflow)
Shift agents away from spreadsheets to a structured, click-driven interface.
* **Assigned Lead Queue:** A clean list of customers to call today, sorted by urgency or callback time.
* **Actionable Lead Cards:** Clicking a lead opens their history, current policy details, and premium amount.
* **Structured Call Logging:**
    * Disposition Dropdown (e.g., No Answer, Call Back, Lost, Renewed).
    * Loss Reason Dropdown (e.g., Competitor, Money Issues, Claim Dissatisfaction).
    * Date/Time picker for scheduling callbacks.
* **Automated Reminders:** System-generated alerts 5 minutes before a scheduled callback.

## 5. Management Analytics Dashboard
Powered by the structured data generated by the Agent CRM.
* **Top-Level KPIs:** Total Gross Written Premium (GWP), Total Commission Earned, and overall Renewal Rate for the current month.
* **Carrier Performance:** Visual breakdown showing which insurance provider yields the highest volume and commission.
* **Agent Performance Tracking:** Conversion rates comparing agents (e.g., Feroz's renewal % vs. Priya's renewal %).
* **Lost Revenue Analysis:** A chart detailing *why* policies were not renewed, utilizing the "Loss Reason" dropdown data from the agents.
* **Upcoming Pipeline:** A widget showing the projected revenue of policies expiring in the next 30/60/90 days.

## 6. Implementation Roadmap
* **Phase 1: DB Architecture & UI Wireframes.** Setup Supabase tables and design the agent/manager interfaces in Figma.
* **Phase 2: The Data Pipeline.** Build the CSV/Excel uploader, the staging grid, and the bulk-insert logic.
* **Phase 3: Agent CRM.** Develop the lead queue, structured call logging, and callback scheduling system.
* **Phase 4: Analytics & Dashboard.** Connect the structured DB data to the charting libraries for management views.
* **Phase 5: Security & Deployment.** Implement role-based access control (RBAC), audit logs (tracking who edited what), CSP, and final testing.
